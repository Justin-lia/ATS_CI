	## Tips: Example of Address Translation by MAP-E(draft3) specification.
	##	e.g. How can the prefix of "2404:7a86:d220:1b00" be translated to CEv4 and CEv6 address?
	##	1) The prefix of "2404:7a86:d220:1b00" is caught in the following MAP-Rule.
	##		--> "ipv6":"2404:7a86:d000::/36","ipv4":"118.110.80.0/20","psid_offset":"4","ea_length":20
	##	2) "2404:7a86:d220:1b00" is devided in to two blocks (36bit+28bit) according to the rule of "2404:7a86:d000::/36".
	##		--> 2404:7a86:d220:1b00 = [24047a86d](36bit)+[2201b00](28bit)
	##	3) ea_bit is extracted from latter "2201b00" using ea_length(20bit).
	##		--> [2201b00](28bit) = [2201b](20bit)+[00](8bit)
	##		--> ea_bit:[2201b]
	##	4) CEv4 is determined by using the rule of "118.110.80.0/20" and the ea_bit.
	##		--> 118.110.80.0/20 = (16x7+6).(16x6+14).(16x5).(0)/20 = [76].[6e].[50].[00]/20 = [766e5](20bit)+[000](12bit)
	##		--> CEv4 is obtained by padding the latter [000](12bit) by ea_bit.
	##			--> [2201b](20bit)=[220](12bit)+[1b](8bit)
	##				--> The rest of ea_bit, [1b](8bit), is the PSID.
	##			--> [766e5](20bit)+[220](12bit)=[766e5220](32bit)=118.110.82.32
	##		--> CEv4:118.110.82.32, PSID:[1b](8bit)
	##	5) CEv6 is determined by using the Prefix, CEv4, and PSID.
	##		--> Interface ID of CEv6 is defined as below.
	##			--> u(0x8bit)+CEv4(32bit)+PSID(16bit:left-padded)+0(8bit)
	##			--> [00](8bit)+[766e5220](32bit)+[001b](16bit)+[00](8bit)=[0076:6e52:2000:1b00]
	##		--> CEv6 is obtained by combining the Prefix and the Interface ID.
	##			--> [2404:7a86:d220:1b00]+[0076:6e52:2000:1b00]=[2404:7a86:d220:1b00:0076:6e52:2000:1b00]
	##		--> CEv6:[2404:7a86:d220:1b00:0076:6e52:2000:1b00]
	##
